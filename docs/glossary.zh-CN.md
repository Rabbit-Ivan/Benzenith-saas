# OpenSpec 术语词表（简体中文）

## 使用规则

- 适用范围：OpenSpec 的 `proposal.md`、`design.md`、`specs/*.md`、`tasks.md`。
- 首次出现建议采用“中文（English）”，后续保持同一写法。
- 下列术语中的英文缩写与关键字保持原文，不进行翻译改写。
- 代码标识符、命令、路径、环境变量始终保持英文原样。

## 核心术语

| English | 推荐中文 | 说明 |
| --- | --- | --- |
| Authentication | 身份认证 | 识别“你是谁”的过程。 |
| Authorization | 授权 | 决定“你能做什么”的过程。 |
| Session | 会话 | 用户登录后在服务端维护的状态。 |
| Access Token | 访问令牌 | 用于访问受保护资源的短期凭证。 |
| Refresh Token | 刷新令牌 | 用于刷新访问令牌的长期凭证。 |
| OAuth | OAuth | 第三方授权协议名，保持英文。 |
| RBAC | RBAC | 基于角色的权限控制模型。 |
| API | API | 接口统称，保持英文。 |
| REST | REST | API 风格术语，保持英文。 |
| GraphQL | GraphQL | API 查询语言，保持英文。 |
| Rate Limit | 速率限制 | 防止请求过载的限制策略。 |
| Audit Log | 审计日志 | 记录关键安全与操作行为的日志。 |
| Least Privilege | 最小权限原则 | 仅授予执行任务所需最小权限。 |

## 认证场景建议用语

- 登录成功：建议使用“登录成功并建立会话”。
- 登录失败：建议区分“凭证错误”“账号状态异常”“风控拦截”。
- 权限拒绝：建议统一使用“无权限访问该资源”。
- 会话失效：建议使用“会话过期，请重新登录”或“令牌无效，请重新认证”。
